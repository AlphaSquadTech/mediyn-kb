# Session Packages

Mediyn lets you offer prepaid session bundles so patients can pay upfront for a set number of sessions.

![Patient Packages](../images/patient/patient-packages.png)

## What You Can Do

- Create prepaid session packages for patients
- Set the number of sessions and total price for each package
- Track how many sessions remain in a package
- View all packages for a patient
- Issue prorated refunds for unused sessions

## Key Concepts

**Session package** — A prepaid bundle of sessions tied to a specific patient and session type. The patient pays once, and sessions are deducted from the package as they are used.

**Session type** — The kind of therapy session the package covers (for example, individual therapy, couples counseling).

**Package stages** — Each package moves through these stages:
- **Pending** — The package has been created but payment has not been confirmed yet
- **Active** — Payment is confirmed and sessions are available for use
- **Exhausted** — All sessions in the package have been used
- **Refunded** — The package was refunded and is no longer valid
- **Expired** — The package has passed its validity period

**Remaining sessions** — The number of unused sessions still available in the package.
