# Tips for Session Recordings & Uploads

## Recommendations

- **Use transcript submission as your primary method** -- Submitting a de-identified transcript is the fastest and most secure way to get session content into Mediyn. It avoids uploading raw recordings entirely.

- **Include pre-generated artifacts when available** -- If your device generates clinical notes locally, submit them along with the transcript. This saves processing time and gives you immediate drafts to review.

- **Check processing status after each session** -- Make it a habit to verify that processing completed successfully. Catching a failed job early means faster turnaround on your clinical notes.

- **Retry failed jobs promptly** -- If a processing job fails, retry it right away. Most failures are temporary and resolve on a second attempt.

- **Use encrypted recordings only as a last resort** -- This option exists for situations where on-device transcription fails entirely. Transcript submission is always preferred.

- **Complete uploads quickly** -- When uploading files, finish the upload before the secure upload window expires. Start the upload promptly after initiating it.

## Common Questions

**Q: What happens if I submit the same transcript twice?**
A: Mediyn recognizes the duplicate and returns the existing result. No duplicate notes are created.

**Q: Can I upload a raw, unencrypted recording?**
A: No. For security reasons, only encrypted recordings are accepted. Use transcript submission as the preferred alternative.

**Q: How long does processing usually take?**
A: Most transcripts are processed within a few minutes. Encrypted recordings may take longer depending on the file size.

**Q: Can I attach documents that are not recordings?**
A: Yes. Use the external document option to attach PDFs, images, or other files to a session.

**Q: What should I do if a job keeps failing after multiple retries?**
A: Contact Mediyn support. There may be an issue with the file format or content that requires assistance.
